# Editorial Design System

로컬 여행 블로그 시안의 디자인 토큰을 정의하는 매뉴얼. 모든 화면 요소(핀·카드·태그·캡션)가 같은 시각 언어로 연결되는 것이 목표다.

## 트리거

디자인 토큰 정의, 태그 팔레트 결정, 카드/핀/배지 시각 언어 통일 시.

## 디자인 토큰 규격

### 컬러
- 웜 뉴트럴 5단: `#FAF9F6 → #EDEAE3 → #C9C4BA → #6B6760 → #1C1A17` (종이 톤, 사진을 죽이지 않음)
- 프라이머리 1색: 잉크 블루 `#2B3A55` (핀·활성 태그·CTA)
- 태그 3색: **무지개 금지**, 미묘한 뉴트럴 변주로 음식/카페/공간 구분
- 핀/라벨 대비 3:1 이상 확보

### 타이포
- 디스플레이: **Fraunces** (하이콘트라스트 세리프)
- 본문: **Manrope**
- 데이터성 텍스트(좌표·주소·최종확인일): **IBM Plex Mono**
- 본문 행간 1.6, 제목 1.15
- 폰트는 CDN 로드

### 레이아웃
- 풀블리드 히어로 + 넉넉한 여백 + 카드 중첩 금지
- 데스크톱 스플릿뷰(좌 글·우 지도), 모바일 접이식 미니맵
- 8배수 그리드
- 모바일 하단 탭 모형, 44px 터치타겟

### 모션
- 차분·전문 200~300ms
- 핀 hover 스프링 미세 스케일
- scrollytelling은 부드러운 페이드/팬 (글리치·바운시 금지)
- `prefers-reduced-motion` 시 scrollytelling 비활성

## 핀 ↔ 카드 ↔ 태그 시각 언어 (경계면 합의)

- 마커 아이콘 = **카테고리 색 + 핀 번호** (naver-map-engineer가 동일 토큰 사용)
- 장소 카드 번호·본문 하이라이트 색 = 핀 번호/색과 1:1 일치
- 태그 배지 색·아이콘·상태(active/default) 스타일은 카드와 동일 팔레트 사용

## 산출물 형식

`artifacts/design/design-tokens.md` — 위 토큰을 CSS 커스텀 프로퍼티 표(`--color-*`, `--font-*`, `--space-*`, `--motion-*`)와 사용 예시로 정리. frontend-builder가 그대로 `:root`에 옮길 수 있게 한다.
