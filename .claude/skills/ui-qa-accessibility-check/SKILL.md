# UI QA & Accessibility Check

작성 역할과 분리된 객관 검증 매뉴얼. 존재 확인보다 **경계면 교차 검증**을 우선하고, 위험 산출물(지도·데이터·index.html)이 끝나는 즉시 점진적으로 점검한다.

## 트리거

index.html 또는 위험 중간 산출물 완료 직후, 최종 통합 전, 부분 재실행 후 영향 범위 점검 시.

## 1. 보안 게이트 (최우선 — 실패 시 즉시 차단)

- [ ] `index.html`·`places.json`·`integration-notes.md` 등 **모든 산출물에 Client Secret 없음**
- [ ] `geocode.mjs`는 Secret을 환경변수(`process.env`)로만 읽음(하드코딩 없음)
- [ ] 스크립트 URL은 `ncpKeyId=YOUR_CLIENT_ID` 자리표시자 또는 사용자 키, Secret 파라미터 없음

Secret이 하나라도 산출물에 박혀 있으면 **자동 통과 금지**, 실패로 보고하고 Orchestrator에 차단 신호.

## 2. 핀 ↔ 카드 ↔ 태그 일치 (경계면)

- [ ] 핀 번호 = 카드 번호 = 본문 하이라이트 색 1:1
- [ ] 마커 아이콘 색 = 카테고리 색 = 태그 배지 색 (디자인 토큰 일치)
- [ ] 좌표(lat/lng)와 카드 주소가 같은 장소를 가리킴(동네 중심에서 과도 이탈 없음)

## 3. 폴백 / 접근성

- [ ] 지도 로드·인증 실패 시 핀 리스트(동등 텍스트) 경로로 모든 장소 접근 가능
- [ ] `navermap_authFailure` + `onerror` 둘 다 폴백 연결
- [ ] `prefers-reduced-motion` 시 scrollytelling 비활성
- [ ] 폴백 시 레이아웃 깨짐 없음

## 4. 반응형 / 밀도

- [ ] 390 / 768 / 1440px에서 텍스트 겹침·표 넘침·차트 깨짐 없음
- [ ] 모바일 단일 컬럼 + 하단 탭 + 44px 터치타겟
- [ ] 섹션별 카드 수·여백 과밀하지 않음

## 5. 단일 페이지 제약

- [ ] 별도 .html(explore/search/saved/article/neighborhood) 없음
- [ ] Explore·Search·Saved·Walks가 drawer·탭·accordion mock으로만 존재

## 6. 성공지표(PRD §3) 점검

| 지표 | 통과 기준 |
|---|---|
| 첫 화면 인지 | 5초 내 도시/동네/탐방 톤 파악 가능한 정보 위계 |
| 지도-글 관계성 | 핀 번호/카드 번호/색상 매칭 명확 |
| 태그 가독성 | 음식/카페/공간 즉시 구분 |
| 반응형 안정성 | 390/768/1440 겹침 없음 |
| 사진/카드 밀도 | 과밀하지 않음 |

## 산출물 형식

`artifacts/qa/qa-report.md` — 항목별 통과/실패, 실패는 위치·근거·수정 제안. 마지막에 종합 판정과 **승인 가능 여부**(`사용 가능` 후보 / `사람 승인 필요` 사유) 명시. 단, 최종 `사용 가능` 전환은 사용자 확인(G1~G4) 후 Orchestrator가 수행한다.

## 품질 체크

- [ ] 보안 게이트를 가장 먼저 실행
- [ ] 존재 확인이 아닌 경계면(핀-카드, 토큰-마커, 좌표-주소) 교차 검증
- [ ] 미검증 영역을 보고서에 명시(예: 실제 키로의 실연동은 사용자 확인 필요)
