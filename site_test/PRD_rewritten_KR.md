# PRD: 동네 탐방 로컬 여행 블로그 — 서울 여러 동네 (단일 페이지 + 실제 지도)

> 재작성일: 2026-06-16 (멀티 동네 개정)
> 원본: `PRD_기본.md` (실험 baseline — 보존)
> 이번 문서의 차이: ① 단일 페이지로 구성, ② 네이버 지도 실제 연동(설정법 명시), ③ API 키는 사용자 프롬프트 입력으로 획득, ④ 키 미설정 시 무료 지도 라이브러리로 지오코딩, ⑤ 임의 더미 장소 → 웹 검색으로 확인한 실제 장소로 교체 후 지오코딩하여 지도에 표시, ⑥ 한국어 줄바꿈·산세리프 한글 웹폰트 규칙 명시, ⑦ **한 동네에 치우치지 않게 서울 여러 동네(서로 다른 구 포함)를 다루고 지도가 여러 구/동에 걸쳐 지오코딩되도록 한다.**

---

## 1. 개요

서울의 **여러 동네**(연희·연남·성수·익선·망원 등, 서로 다른 구에 걸침)를 직접 걸으며 기록하는 동네 탐방 블로그. 음식점·카페·공간·골목을 글과 사진으로 소개하고, **실제 지도 위에 장소를 핀으로 표시**한다. 모든 콘텐츠는 별도 페이지 없이 **하나의 페이지(`index.html`)** 안에서 스크롤·탭·드로어로 제공한다.

> 설계 원칙: 특정 동네에 장소가 몰리지 않도록 **동네별로 비슷한 수(3곳 내외)** 를 유지하고, 동네는 손쉽게 추가/교체할 수 있는 데이터로 둔다. 새로운 동네는 `data.js`에 도로명주소만 추가하면 같은 지오코딩 파이프라인으로 지도에 표시된다.

## 2. 목적

- 작성자가 다닌 동네/장소를 한 페이지에서 기록·공유한다.
- 방문객이 동네 단위로 좋은 장소를 빠르게 찾도록 한다.
- 지도와 글을 한 화면에서 나란히 보여 "어디에 있는지"를 즉시 전달한다.

## 3. 대상 사용자

- 반나절 동네 코스를 찾는 도시 직장인
- 카페·맛집·골목 분위기를 사진/톤으로 고르는 사람
- 방문 전 동네 단위로 리서치하는 사람

## 4. 단일 페이지 구성 (페이지 분할 금지)

`explore.html`, `article.html`, `neighborhood.html` 같은 별도 공개 페이지를 만들지 않는다. 아래 섹션을 한 페이지 안에 둔다.

1. **히어로**: 도시/동네 제목, 한 줄 소개, 대표 태그, **추천 동네 칩(여러 동네)**.
2. **지도 + 장소 스플릿뷰**: 좌(데스크톱) 또는 상단(모바일)에 실제 지도, 우/하단에 번호가 매겨진 장소 카드 리스트. **핀 번호 = 카드 번호 = 본문 하이라이트 색**으로 연결. 지도는 여러 구/동의 핀을 모두 담도록 지오코딩 후 `fitBounds`로 범위를 자동 조정한다.
3. **동네 아티클**: 다루는 **각 동네마다** 짧은 에디토리얼 글 + 그 안에 장소 인라인 칩(동네별 균형 유지).
4. **카테고리/태그 필터**: 카페 · 맛집 · 공간(시장/공원) 필터(지도 핀과 카드 동시 필터).
5. **검색**: 장소명 검색(현재 데이터셋 내 클라이언트 사이드 필터).
6. **소개(About)**: 블로그/작성자 소개 + 데이터 출처.

## 5. 지도 연동 — 네이버 지도 (기본 경로)

### 5.1 키 발급 절차 (네이버 클라우드 플랫폼)

네이버 지도 Maps API는 **네이버 클라우드 플랫폼 콘솔**에서 발급한다. (공공/금융 콘솔은 개인·일반 기업 콘솔로 통합되어, 신규 `ncpKeyId` 발급이 필요하다.)

1. 네이버 클라우드 플랫폼 콘솔 → **Services → Application Service → Maps** 진입.
2. **Application 등록** → 사용할 서비스로 **Web Dynamic Map** 선택. 주소↔좌표 변환을 쓰므로 **Geocoding API**도 함께 선택.
3. **Web 서비스 URL**에 사이트가 열릴 도메인(로컬 테스트 시 `http://localhost:포트`)을 등록한다. (등록 안 된 도메인에서 호출하면 인증 실패)
4. 발급된 **Client ID = `ncpKeyId`** 값을 확인한다.

### 5.2 스크립트 로드 (Geocoder 서브모듈 포함)

```html
<script
  src="https://oapi.map.naver.com/openapi/v3/maps.js?ncpKeyId=YOUR_CLIENT_ID&submodules=geocoder">
</script>
```

- 인증 파라미터는 `ncpKeyId` (구 `ncpClientId`가 아님 — 업그레이드 후 명칭 변경).
- 주소→좌표 변환에 `submodules=geocoder`가 필요하다.

### 5.3 지도 초기화

```javascript
const map = new naver.maps.Map('map', {
  center: new naver.maps.LatLng(37.5650, 126.9270), // 연남·연희 중간
  zoom: 15
});
```

### 5.4 지오코딩 (주소 → 좌표 → 마커)

```javascript
naver.maps.Service.geocode({ query: '서울 서대문구 연희로 135' }, (status, res) => {
  if (status !== naver.maps.Service.Status.OK) return; // 호출마다 일일 허용량 소비 → 상태 확인 필수
  const a = res.v2.addresses[0];
  if (!a) return;
  const lat = parseFloat(a.y); // y = 위도
  const lng = parseFloat(a.x); // x = 경도
  new naver.maps.Marker({ position: new naver.maps.LatLng(lat, lng), map, title: a.roadAddress });
});
```

### 5.5 인증 실패 처리

```javascript
window.navermap_authFailure = function () {
  // 키 오류/도메인 미등록/허용량 초과 → 무료 지도 경로로 자동 전환
  loadFreeMapFallback();
};
```

### 5.6 API 키 획득 방식 — 사용자 프롬프트 입력

- 하드코딩 금지. 페이지에서 **입력 UI**로 `ncpKeyId`를 받는다.
- 동작:
  - 입력값이 있으면 위 스크립트를 동적으로 주입 → 네이버 지도 + 지오코딩 사용. 값은 `localStorage`에 저장(다음 방문 시 재사용), "키 지우기" 제공.
  - **입력값을 비우거나 "무료 지도로 보기"를 선택하면 → 6장의 무료 경로로 전환**한다.
  - 인증 실패(`navermap_authFailure`) 시에도 자동으로 무료 경로로 폴백한다.

> **여러 구/동 지원**: 장소가 서로 다른 구(서대문·마포·성동·종로 등)에 걸쳐 있어도 지오코딩은 각 도로명주소를 그대로 변환한다. 네이버 경로는 `naver.maps.Service.geocode`, 무료 경로는 Nominatim에 동일한 주소 문자열을 넘기며, 최종적으로 모든 핀을 담도록 지도 범위를 자동(`fitBounds`)으로 맞춘다.

## 6. 지도 연동 — 무료 폴백 (키 미설정 시)

API 키를 설정할 수 없을 때는 **키가 필요 없는 무료 스택**으로 지오코딩·표시한다.

- **지도/타일**: Leaflet + OpenStreetMap 타일 (키 불필요).
- **지오코딩**: OpenStreetMap **Nominatim** (`https://nominatim.openstreetmap.org/search?format=json&q=<도로명주소>`, 키 불필요).
- **사용 정책 준수**: 요청을 순차 처리하고 호출 간 약 1초 간격을 둔다(과도한 동시 호출 금지). 결과 좌표로 Leaflet 마커를 찍는다.
- 무료 경로와 네이버 경로는 **동일한 장소 데이터(7장, 15개 장소 · 5개 동네)** 를 입력으로 쓰고, 핀↔카드 연결·필터 동작은 두 경로에서 같게 유지한다.

> 지오코딩 실패 장소는 지도 핀이 없어도 카드 리스트에는 그대로 노출한다(지도의 동등 텍스트 경로 보장).

## 8. 디자인 방향

- 에디토리얼·사진 중심, 차분한 톤. 무지개 팔레트 금지(뉴트럴 + 포인트 1색).
- 지도와 글이 한 화면에서 어우러지는 스플릿 레이아웃, 8배수 간격.
- 카드 중첩 금지, 넉넉한 여백.

### 8.1 글꼴 (산세리프 한글 웹폰트 — 튀지 않게)

- **본문·제목 모두 산세리프 한글 웹폰트로 통일**한다. 디스플레이 세리프(예: Fraunces)처럼 튀는 글꼴은 쓰지 않는다.
- 1순위: **Pretendard**(또는 Pretendard Variable) — 한글/라틴 균형이 좋고 본문 가독성이 높은 무료 웹폰트.
- 폴백 스택: `Pretendard, -apple-system, "Apple SD Gothic Neo", "Noto Sans KR", system-ui, sans-serif` — 웹폰트 로딩 전후 레이아웃 흔들림(FOUT)을 줄인다.
- 데이터성 텍스트(주소·최종확인일)만 모노 폰트를 절제해서 쓸 수 있으나, 한글 본문엔 쓰지 않는다.

### 8.2 한국어 줄바꿈 (글씨체 영향 없이 자연스럽게)

- 한글 텍스트 블록에 **`word-break: keep-all`** 을 적용해 **단어 중간에서 줄이 끊기지 않게** 한다.
- 긴 URL/영문 토큰만 `overflow-wrap: anywhere`로 예외 처리한다.
- 제목은 의미 단위로 끊기도록 `<br>` 남용 대신 컨테이너 폭으로 제어하고, `text-wrap: pretty`(지원 브라우저)로 고아 단어를 줄인다.
- 본문 행간 1.6~1.7, 제목 1.2~1.3. 자간은 본문 0(과한 자간 금지).
- 글꼴 교체로 줄 수가 바뀌어 어색해지지 않도록, 웹폰트와 시스템 폴백의 메트릭이 크게 다르지 않은 폰트만 쓴다.

## 9. 기술 스택

- 정적 **HTML / CSS / JS 단일 페이지**(빌드 불필요). 외부 콘텐츠 CMS·백엔드 없음.
- 지도: 네이버 Maps JS v3(+geocoder) **또는** Leaflet+OSM+Nominatim(무료 폴백).
- 폰트: Pretendard (CDN).

## 10. 비기능 요건

- 모바일 대응(390 / 768 / 1440 뷰포트에서 겹침 없음, 터치 타깃 44px+).
- 텍스트 대비 WCAG AA 근접(본문 4.5:1).
- `prefers-reduced-motion` 존중.
- 지도/지오코딩 호출 실패·키 부재 등 모든 경우에 **카드 리스트는 항상 보인다**(점진적 향상).

## 11. 수용 기준 (핵심)

1. 키 없이 첫 진입 시 무료 지도(Leaflet+OSM)에서 실제 15개 장소가 지오코딩되어 핀으로 보인다(5개 동네 × 3곳, 서로 다른 4개 구에 걸침, 카페·맛집·공간 혼합).
2. `ncpKeyId` 입력 시 네이버 지도로 전환되고 같은 장소가 핀으로 보인다. 인증 실패 시 무료 지도로 자동 폴백한다.
3. 핀과 카드가 번호·색으로 연결되고, 카테고리 필터가 핀·카드에 동시 적용된다.
4. 한글 본문이 단어 중간에서 끊기지 않는다(`keep-all` 확인).
5. 별도 공개 HTML 페이지 없이 한 페이지 안에서 모든 섹션이 동작한다.
